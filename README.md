# Neural Geometry ‚Äî Multi-Engine AI Research Visualization

A cross-engine portfolio project demonstrating **Godot 4, Defold, Solar2D, Panda3D, and Stride** as rendering backends for AI research asset pipelines.

## What It Does

A simulated VAE decoder (pure NumPy MLP) traverses a 4D latent space along a Lissajous curve. Each of the 60 animation frames maps a latent vector to per-vertex displacements on an icosphere (642 vertices, 1280 faces). The resulting animated geometry and activation-map textures are consumed by all five engines.

**What you see in each engine:**
- A sphere that continuously morphs shape
- Per-vertex colors shift blue (low displacement) ‚Üí orange (high) ‚Äî a displacement heatmap
- An activation-map texture cycling on the surface (generated from the latent vector)
- HUD: current frame, latent vector `z = [z0, z1, z2, z3]`, inference time

**AI research relevance:** Drop in a real VAE/GAN decoder in `pipeline/neural_sim.py` `NeuralDecoder.decode()` ‚Äî game engines become real-time inference visualizers at zero added cost.

## Quick Start

### 1. Install Python dependencies
```bash
pip3 install -r requirements.txt
```

### 2. Generate all shared assets (run once)
```bash
python3 pipeline/run_pipeline.py
```
This writes ~13 MB of `.npy`, `.glb`, `.bam`, `.png`, and `.json` files into `shared_assets/`.

### 3. Verify pipeline output
```bash
python3 tools/verify_pipeline.py
```

### 4. Run Panda3D demo (immediate, no editor needed)
```bash
python3 engines/panda3d/main.py
```
**Controls:** `F1` = toggle GLSL heatmap shader | `R` = reset | `Escape` = quit

## 2D Engine Strategy (Defold & Solar2D)

Rather than fighting 2D-first engines with 3D, we use a **radial projection** ‚Äî each vertex's displacement magnitude is mapped to its distance from center along a radial angle. This produces a "neural fingerprint" ‚Äî a circular pattern unique to each latent vector, closely resembling how researchers visualize high-dimensional data (radar charts, star plots).

The activation map background uses the same latent vector data, so background texture and foreground geometry are visibly correlated ‚Äî a viewer can see them change together.

## Data Flow

```
neural_sim.py ‚Üí 60-frame Lissajous latent sequence (60, 4)
    ‚Üí NeuralDecoder MLP ‚Üí displacement magnitudes (60, 642)
        ‚Üí mesh_gen ‚Üí vertex positions  keyframes/frame_NNN.npy  (60, 642, 3)
        ‚Üí mesh_gen ‚Üí vertex colors     vertex_colors.npy         (60, 642, 4)
        ‚Üí texture_gen ‚Üí activation PNGs + spritesheet
        ‚Üí exporters ‚Üí .glb (Godot/Stride) | .bam (Panda3D) | JSON (Defold/Solar2D)
```

**.npy as interchange format:** trivially-specified binary (6-byte magic + ASCII header + raw float32). Native in Python; ~30-line parsers in GDScript and C#; pre-decoded to JSON for Lua engines.

## Swapping in Real AI

Replace the simulated decoder with real inference:

```python
# pipeline/neural_sim.py ‚Äî NeuralDecoder.decode()

# Before (simulated):
def decode(self, z):
    x = z.copy()
    for w, b in zip(self.weights[:-1], self.biases[:-1]):
        x = np.tanh(x @ w + b)
    return (np.tanh(x @ self.weights[-1] + self.biases[-1]) + 1.0) * 0.5

# After (real PyTorch model):
def decode(self, z):
    import torch
    with torch.no_grad():
        z_tensor = torch.from_numpy(z).float().unsqueeze(0)
        out = self.model.decode(z_tensor).squeeze(0).numpy()
    return (np.tanh(out) + 1.0) * 0.5
```

Then re-run `python3 pipeline/run_pipeline.py` ‚Äî all five engines automatically get new geometry.
